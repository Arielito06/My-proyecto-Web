Procesamiento en el Servidor Web

¿Qué ocurre en el servidor?

Cuando el servidor recibe la solicitud HTTP enviada por el navegador, su trabajo es interpretar qué se le pide. Primero revisa la URL y el método (GET, POST, etc.), luego localiza el recurso solicitado (puede ser un archivo HTML, una imagen, o un programa que genera contenido dinámico). Si el recurso existe, el servidor lo prepara y lo envía de vuelta junto con cabeceras que indican el tipo de contenido, el tamaño, y el código de estado. Todo esto ocurre en milisegundos.

¿Qué hace el servidor?

1Recibe la petición HTTP

El servidor escucha en el puerto 80 (HTTP) o 443 (HTTPS) y recibe la solicitud del navegador.

2Identifica el recurso

Localiza el archivo o recurso solicitado (HTML, CSS, imagen, etc.) en el sistema de archivos.

3Ejecuta código (si es dinámico)

Si es una aplicación dinámica, ejecuta PHP, Python, Node.js u otro lenguaje para generar contenido.

4Genera respuesta

Prepara el contenido y añade cabeceras HTTP antes de enviarlo al navegador.

Ejemplo de encabezado de respuesta

HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1250
Server: Apache/2.4

Códigos de estado comunes

Los códigos de estado HTTP son números de 3 dígitos que el servidor devuelve al navegador para decir qué pasó con su solicitud.

200 OK

Todo correcto, el servidor envía la página

404 Not Found

Recurso no existe (página no encontrada)

500 Internal Server Error

Error interno del servidor
